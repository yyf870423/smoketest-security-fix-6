## 技术决策

### Dockerfile 设计
- 基础镜像：`node:20-alpine`（生产优化，体积小）
- 多阶段构建：
  - Stage 1 (builder)：安装依赖（本项目无 npm 依赖，此阶段主要为未来扩展性预留）
  - Stage 2 (production)：仅复制运行所需文件
- 非 root 用户：使用 `node` 用户（alpine 镜像内置）
- 健康检查：`wget -qO- http://localhost:3000/count` 每 30s 检查一次
- 由于无 npm 外部依赖，构建非常简单

### docker-compose.yml 设计
- 仅需一个 `app` 服务（无数据库/缓存依赖）
- 端口：`${APP_PORT:-3000}:3000`
- 环境变量：`DEPLOY_ENV=${DEPLOY_ENV:-development}`
- 健康检查继承自 Dockerfile
- 数据卷：由于是无状态计数器，不需要持久化存储（计数在内存中）

## 备注

- 本项目无 npm install 步骤（package.json 无 dependencies 字段）
- 无需数据库或缓存服务
- docker-compose.yml 中添加 `restart: unless-stopped` 策略
