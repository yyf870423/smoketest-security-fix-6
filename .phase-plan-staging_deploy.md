# Phase Plan: staging_deploy（容器化部署验证）

## 阶段目标

确保项目具备容器化部署能力：验证 Dockerfile、docker-compose.yml、.dockerignore 均存在且正确，并完成完整的构建 → 启动 → 健康检查 → 清理流程。

## 项目分析结果

- **类型**：Node.js HTTP 应用（无框架，使用内置 `http` 模块）
- **入口**：`src/index.js`
- **端口**：`process.env.PORT || 3000`
- **外部依赖**：无（无 node_modules，无数据库，无缓存）
- **启动命令**：`node src/index.js`
- **健康检查端点**：`GET /count`

## 步骤清单

- [x] 步骤 1：验证 Dockerfile 与当前项目结构匹配（入口、端口、健康检查端点）
- [x] 步骤 2：验证 docker-compose.yml 完整性（环境变量端口、DEPLOY_ENV、健康检查、资源限制）

- [x] 步骤 5：运行 `docker compose up -d` 确保容器启动成功

- [x] 步骤 7：运行 `docker compose down` 清理环境

## 回滚方案（本阶段）

- 若 docker build 失败 → 检查 Dockerfile 语法和基础镜像可用性，修复后重试
- 若容器启动失败 → `docker compose logs app` 查看日志，排查端口冲突或配置问题
- 若健康检查不通过 → 检查应用是否监听正确端口，wget 是否可用
